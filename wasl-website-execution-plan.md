# خطة تنفيذ موقع وصل لنمو الأعمال

## 1. الهدف العام

بناء موقع تعريفي احترافي لشركة **وصل لنمو الأعمال** باستخدام `HTML` و`CSS` و`JavaScript` فقط، يعرض الشركة كجهة تسويقية استراتيجية لا تقدم خدمات متفرقة، بل تبني رحلة متكاملة تحول الفكرة إلى علامة، والعلامة إلى أثر.

الموقع يجب أن يشعر الزائر منذ أول شاشة أنه داخل تجربة علامة راقية، واثقة، عربية، عميقة، ومختلفة عن المواقع التسويقية التقليدية.

## 2. شخصية الموقع

- النبرة: عميقة، ملهمة، استراتيجية، راقية، وواضحة.
- الانطباع البصري: فخم، متزن، حركي، إبداعي، ومبني حول فكرة الرحلة.
- الرسالة الأساسية: "من الفكرة إلى النمو" و"الفكرة تتحول إلى أثر عندما تجد طريقها الصحيح".
- الجمهور المستهدف: أصحاب المشاريع، الشركات الناشئة، العلامات القائمة التي تريد إعادة بناء حضورها، ورواد الأعمال الباحثون عن شريك تسويقي متكامل.

## 3. الهوية البصرية

### الألوان

استخدام ألوان العميل كما هي مع تنظيمها داخل CSS Variables:

```css
:root {
  --primary: #002889;
  --primary-dark: #04217C;
  --accent: #EDB75C;
  --white: #FCFCFD;
  --light: #97A8D1;
  --secondary: #536CAF;
  --ink: #07142F;
  --muted: #D9E0F5;
}
```

### طريقة استخدام اللون

- الأزرق الملكي يكون خلفية المشاهد الكبرى، خاصة الـ hero وأقسام الرحلة.
- الذهبي يستخدم كإشارة توجيهية، مثل نقاط المسار، أرقام المراحل، الخطوط الدقيقة، وحالات hover.
- الأبيض يستخدم للنصوص والمساحات الهادئة.
- الرمادي المزرق يستخدم للحدود الناعمة، الظلال، النصوص الثانوية، والعمق البصري.
- تجنب جعل الصفحة كلها زرقاء. يجب إدخال مقاطع بيضاء، ومقاطع داكنة، ومقاطع بتدرجات دقيقة حتى تبقى التجربة غنية.

### الشعار

- استخدام الصورة `wasl.jpg` داخل المشروع.
- في الهيدر يفضل استخدام نسخة مقصوصة أو موضوعة داخل مساحة مربعة صغيرة، مع اسم الشركة نصيا بجانبها: "وصل لنمو الأعمال".
- في الـ hero يمكن استخدام الشعار كعنصر بصري كبير داخل المشهد، لكن بدون تكرار مزعج.
- يجب الحفاظ على حضور الشعار في أول شاشة بوضوح.

## 4. الاتجاه والخطوط

الموقع عربي بالكامل، لذلك:

- تعيين `dir="rtl"` و`lang="ar"` في وسم `html`.
- استخدام خط عربي احترافي من Google Fonts مثل:
  - `IBM Plex Sans Arabic` للنصوص.
  - `Cairo` أو `Tajawal` للعناوين في حال الحاجة لطابع أكثر حداثة.
- ضبط أحجام الخطوط باستعمال `rem` و`clamp()` بشكل محدود للعناوين فقط، مع عدم الاعتماد على عرض الشاشة وحده.
- زيادة `line-height` للنصوص الطويلة إلى `1.9` تقريبا لضمان قراءة مريحة.

## 5. هيكل الملفات

```text
wasl/
  index.html
  assets/
    images/
      wasl.jpg
  css/
    style.css
  js/
    main.js
```

إن كان التنفيذ داخل نفس المجلد الحالي، يمكن إبقاء `wasl.jpg` في الجذر أو نقله إلى `assets/images/wasl.jpg` مع تعديل المسار في HTML.

## 6. بنية الصفحة المقترحة

### 6.1 الهيدر

مكونات الهيدر:

- الشعار واسم الشركة.
- روابط تنقل داخلية:
  - البداية
  - قصتنا
  - رحلتك مع وصل
  - منظومة وصل
  - خدماتنا
  - ابدأ رحلتك
- زر CTA واضح: "ابدأ رحلتك".

سلوك الهيدر:

- ثابت أعلى الصفحة عند التمرير.
- خلفيته شفافة في أول الشاشة، ثم تتحول إلى أزرق داكن شبه زجاجي عند التمرير.
- على الجوال يتحول إلى زر قائمة بأيقونة، والقائمة تنزلق من اليمين.

### 6.2 مشهد البداية Hero

الفكرة الإبداعية:

تصميم أول شاشة كمشهد "رحلة بين الفكرة والأثر". لا يكون مجرد عنوان وزر، بل مساحة حية فيها خط مسار ذهبي متحرك يربط بين كلمات أو نقاط:

`الفكرة` ← `الوضوح` ← `الهوية` ← `القصة` ← `الوصول` ← `الأثر`

المحتوى:

- عنوان رئيسي: `الفكرة × الأثر`
- جملة بارزة: `الأثر الذي يبقى… لا يأتي صدفة.`
- نص داعم:
  `وبين الفكرة والأثر… هناك رحلة. رحلة تبدأ بفهم عميق، وتتشكل باستراتيجية واضحة، وتكتسب هويتها، وتُروى قصتها، وتصل إلى جمهورها، وتستمر بمنظومة ذكية.`
- CTA أساسي: `ابدأ رحلتك`
- CTA ثانوي: `اكتشف منظومة وصل`

تفاصيل UI:

- خلفية أزرق ملكي بتدرج عميق.
- الشعار يظهر كعنصر مضيء أو بارز في جهة، مع ظل ناعم.
- خط ذهبي متحرك SVG أو CSS يرسم مسار الرحلة عند تحميل الصفحة.
- كلمات الرحلة تظهر بتتابع خفيف.
- لا تستخدم بطاقات داخل الـ hero. النص يكون فوق الخلفية مباشرة.

### 6.3 شريط وعد العلامة

قسم قصير بعد الـ hero مباشرة، يثبت الرسالة:

`في وصل، لا نقدم خدمات منفصلة… بل نبني رحلة متكاملة تُحوّل الأفكار إلى علامات، والعلامات إلى أثر.`

التصميم:

- خلفية بيضاء.
- الجملة تظهر بتكوين طباعي قوي، مع تمييز الكلمات `الأفكار` و`علامات` و`أثر` بالذهبي أو الأزرق.
- يمكن إضافة خط أفقي ذهبي رفيع يمر خلف الكلمات كامتداد لفكرة الرحلة.

### 6.4 قسم قصتنا

المحتوى:

استخدام نص "قصتنا" من ملف العميل مع تقصيره بصريا إلى فقرات منظمة. لا يجب وضع النص ككتلة طويلة واحدة.

التركيب المقترح:

- عنوان: `قصتنا`
- مدخل: `من مكانٍ لم يرَ فيه الآخرون القيمة… بدأت الحكاية.`
- عرض الاقتباسات الثلاثة بشكل بصري داخل أعمدة أو شرائط:
  - `هذا التخصص لن يصنع فرقًا.`
  - `مجالكم مجرد أدوات.`
  - `لا مستقبل حقيقي لما تفعلون.`
- بعدها يتم عرض التحول:
  `لكننا كنا نرى شيئًا مختلفًا.`
- ختام القسم:
  `ومن هنا بدأت وصل… رحلة صناعة الأثر.`

الفكرة الإبداعية:

القسم يكون مثل "مشهد ذاكرة". عند التمرير تظهر الجمل الناقدة بلون باهت، ثم تنسحب بصريا لتحل مكانها جملة وصل الإيجابية. يمكن تنفيذ ذلك بـ CSS transitions وIntersectionObserver.

### 6.5 قسم رحلتك مع وصل

هذا القسم يمثل العمود الفقري للصفحة.

المراحل:

1. `نكتشف الحكاية`
2. `نبني شخصيتك`
3. `نروي قصتك`
4. `نفتح طريق الوصول`
5. `نبني ما يستمر`

التصميم المقترح:

- Timeline عمودي على سطح المكتب، لكنه غير تقليدي.
- خط ذهبي متعرج أو منحني يمر بين المراحل.
- كل مرحلة تظهر كنقطة في المسار مع رقم كبير وخلفية متغيرة.
- عند الوصول لمرحلة معينة بالتمرير، تضيء نقطتها ويتغير نص جانبي كبير.
- على الجوال تتحول إلى بطاقات متتابعة كاملة العرض مع رقم المرحلة في الأعلى.

محتوى كل مرحلة:

- عنوان المرحلة.
- جملة افتتاحية قصيرة.
- فقرة شرح.
- جملة ختامية عميقة من نص العميل.

مثال للمرحلة الأولى:

- العنوان: `نكتشف الحكاية`
- الجملة: `كل مشروع يبدأ بقصة… ونحن نبدأ بالاستماع.`
- الشرح: `نفهم مشروعك، رؤيتك، أهدافك، والتحديات التي تواجهها. نبحث في جمهورك والسوق، ونكتشف الفرصة والقصة التي تستحق أن تظهر.`
- الخاتمة: `لأن الطريق الصحيح يبدأ عندما نعرف أين نقف.`

### 6.6 قسم منظومة وصل

الغرض:

تقديم المنتجات أو المراحل الكبرى التي يمكن أن يشتريها العميل أو يدخل منها إلى خدمات وصل.

العناصر:

1. `اكتشف بوصلتك`
2. `اصنع حضورك`
3. `ابنِ منظومة أثرك`

التصميم:

- ليس بطاقات تقليدية فقط. الأفضل تنفيذ القسم كتجربة tabs أو panels.
- في الجانب الأيمن قائمة الثلاث مراحل.
- في الجانب الأيسر تفاصيل المرحلة المختارة.
- عند الضغط على مرحلة، يتحرك مؤشر ذهبي، ويتغير المحتوى بسلاسة.
- يمكن استخدام رمز بصري لكل مرحلة:
  - بوصلة للاتجاه.
  - موجة أو إشارة للحضور.
  - شبكة نقاط للمنظومة.

محتوى كل panel:

- عنوان.
- وصف.
- قائمة نقاط.
- "هذه المرحلة تناسبك إذا كنت..." بثلاث نقاط.
- CTA صغير: `ناقش هذه المرحلة معنا`.

### 6.7 قسم خدماتنا

الخدمات الخمس:

1. `نعرف الاتجاه`
2. `نبني الهوية`
3. `نصنع القصة`
4. `نوصل الرسالة`
5. `نبني الأثر`

التصميم:

- شبكة خدمات مميزة، لكن ليست بطاقات مملة.
- كل خدمة تظهر كـ service block واسع يحتوي:
  - الرقم.
  - العنوان.
  - وصف قصير.
  - قائمة مخرجات الخدمة.
  - خط ذهبي أو تفصيل بصري مستوحى من الشعار.
- يمكن جعل الخدمة الأولى والثانية في صف، والثالثة بعرض أكبر، والرابعة والخامسة في صف، لإعطاء إيقاع بصري متغير.

مخرجات الخدمة:

- `نعرف الاتجاه`: تخطيط استراتيجي، دراسة السوق، تحديد الجمهور، بناء الرسالة.
- `نبني الهوية`: الشعار، الألوان، الخطوط، أسلوب التواصل، النظام البصري.
- `نصنع القصة`: كتابة المحتوى، السيناريو، إنتاج الفيديو، التصوير، الموشن جرافيك، التعليق الصوتي، المونتاج.
- `نوصل الرسالة`: إدارة الحملات، الإعلانات، تحسين الأداء، استراتيجيات الوصول.
- `نبني الأثر`: SEO، صفحات الهبوط، تطبيقات الجوال، أتمتة الأعمال، الأنظمة الرقمية.

### 6.8 قسم CTA النهائي

المحتوى:

- عنوان: `هل فكرتك جاهزة لتصل؟`
- نص: `ابدأ رحلتك مع وصل، ودعنا نبني الطريق من الفكرة إلى الأثر.`
- زر: `ابدأ رحلتك`
- زر ثانوي: `تواصل معنا`

التصميم:

- خلفية أزرق داكن.
- مسار ذهبي ينتهي عند الزر.
- ظهور الشعار بشكل خافت أو كبير في الخلفية.
- يجب أن يكون القسم حاسما وبسيطا.

### 6.9 الفوتر

المحتوى:

- شعار وصل.
- جملة: `من الفكرة إلى النمو`.
- روابط داخلية.
- بيانات تواصل placeholders لحين توفيرها:
  - البريد الإلكتروني
  - رقم الهاتف
  - واتساب
  - إنستغرام
  - لينكدإن
- حقوق النشر: `© 2026 وصل لنمو الأعمال. جميع الحقوق محفوظة.`

## 7. التفاعلات والحركة

استخدام JavaScript بشكل مدروس لإضافة تجربة راقية:

- `IntersectionObserver` لإظهار الأقسام عند التمرير.
- تفعيل المرحلة الحالية في Timeline.
- تبديل محتوى "منظومة وصل" عبر tabs.
- فتح وإغلاق قائمة الجوال.
- تغيير حالة الهيدر بعد التمرير.
- تحريك أرقام أو كلمات مختارة عند ظهورها.
- تنفيذ parallax خفيف جدا للشعار أو عناصر المسار في الـ hero.

قيود مهمة:

- الحركة يجب أن تكون راقية وهادئة، لا مزعجة.
- دعم `prefers-reduced-motion` وإيقاف الحركات الكبيرة لمن يفضل تقليل الحركة.
- عدم الاعتماد على مكتبات خارجية إلا إن دعت الحاجة. يمكن تنفيذ المطلوب بـ vanilla JS.

## 8. أفكار تصميم إبداعية أساسية

### فكرة المسار الذهبي

يكون الموقع كله مبنيا حول "طريق يصل الفكرة بالأثر". يظهر المسار في:

- الـ hero كخط حي.
- قسم الرحلة كـ timeline.
- أزرار CTA كخط يصل إليها.
- الفواصل بين الأقسام.

### فكرة التحول

كل قسم يعرض انتقالا:

- من التشكيك إلى الرؤية في "قصتنا".
- من الحكاية إلى الهوية في "رحلتك".
- من الحضور إلى المنظومة في "منظومة وصل".
- من الخدمات إلى الأثر في CTA النهائي.

### فكرة الشعار كعنصر معماري

استخدام شكل الشعار أو زواياه كمصدر للإلهام:

- قصات هندسية بسيطة.
- علامات ذهبية صغيرة.
- إطارات غير مكتملة.
- أسهم أو مربعات مائلة مستوحاة من الرمز الذهبي أعلى الشعار.

## 9. متطلبات UI/UX

- أول شاشة يجب أن تشرح قيمة وصل خلال 5 ثوان.
- كل قسم يجب أن يجيب على سؤال واضح:
  - من أنتم؟
  - لماذا وُجدتم؟
  - كيف تعملون؟
  - ماذا تقدمون؟
  - كيف أبدأ؟
- لا تكديس للنصوص الطويلة. تقطيع المحتوى إلى فقرات قصيرة وعناوين فرعية.
- زر CTA يجب أن يظهر في أول الشاشة، وسط الصفحة تقريبا، وفي النهاية.
- تجربة الجوال يجب أن تكون مصممة بذاتها، لا مجرد تصغير لسطح المكتب.
- كل النصوص يجب أن تكون قابلة للقراءة بوضوح على الخلفيات الداكنة.
- عدم استخدام زخارف كثيرة بلا وظيفة.

## 10. متطلبات HTML

- استخدام عناصر semantic:
  - `header`
  - `nav`
  - `main`
  - `section`
  - `article`
  - `footer`
- إضافة `aria-label` للأزرار والأيقونات.
- أزرار الروابط الداخلية تستخدم `href="#section-id"`.
- الصور تحتوي على `alt` مناسب.
- ترتيب العناوين يبدأ من `h1` واحد فقط في الـ hero، ثم `h2` للأقسام.

## 11. متطلبات CSS

- بناء نظام CSS واضح:
  - Variables
  - Reset بسيط
  - Typography
  - Layout
  - Components
  - Sections
  - Responsive
  - Motion
- استخدام `max-width` للمحتوى: بين `1120px` و`1240px`.
- استخدام `container` عام.
- استخدام `grid` للأقسام المركبة.
- استخدام `flex` للهيدر والأزرار.
- ضبط نقاط التجاوب:
  - 480px
  - 768px
  - 1024px
  - 1280px
- عدم استخدام palette أحادية بالكامل. يجب كسر الأزرق بمساحات بيضاء وذهبية ورمادية مزرقة.

## 12. متطلبات JavaScript

الملف `main.js` يحتوي وحدات صغيرة واضحة:

- `initHeaderScroll()`
- `initMobileMenu()`
- `initRevealOnScroll()`
- `initJourneyTimeline()`
- `initSystemTabs()`
- `initSmoothAnchors()`

يجب تجنب كتابة منطق معقد أو الاعتماد على DOM غير مستقر.

## 13. تحسين الأداء

- ضغط الصورة `wasl.jpg` أو إنشاء نسخة WebP إن أمكن.
- استخدام `loading="lazy"` للصور غير الموجودة في أول شاشة.
- تقليل الظلال الثقيلة.
- عدم استخدام فيديو خلفي.
- عدم تحميل مكتبات حركة كبيرة.
- تحميل الخطوط بأوزان محدودة فقط: 400، 600، 700.

## 14. تحسين SEO

العنوان المقترح:

`وصل لنمو الأعمال | من الفكرة إلى النمو`

وصف الصفحة:

`وصل لنمو الأعمال شركة تسويق تبني رحلة متكاملة من الاستراتيجية والهوية وصناعة المحتوى إلى الحملات والمنظومات الرقمية المستدامة.`

كلمات محورية طبيعية داخل المحتوى:

- شركة تسويق
- نمو الأعمال
- هوية بصرية
- استراتيجية تسويقية
- صناعة محتوى
- حملات رقمية
- صفحات هبوط
- أتمتة الأعمال
- تحسين محركات البحث

## 15. قائمة تنفيذ عملية

1. تجهيز هيكل الملفات.
2. نقل الشعار إلى مجلد الصور أو تثبيت مساره.
3. بناء HTML كامل بالأقسام المذكورة.
4. إضافة CSS variables ونظام الخطوط.
5. تصميم الـ hero والمسار الذهبي.
6. تصميم قسم القصة بتقسيم بصري واضح.
7. بناء timeline رحلتك مع وصل.
8. بناء tabs لمنظومة وصل.
9. بناء شبكة الخدمات.
10. بناء CTA النهائي والفوتر.
11. إضافة JavaScript للتفاعلات.
12. اختبار responsive على الجوال والتابلت وسطح المكتب.
13. اختبار القراءة والتباين.
14. اختبار الروابط الداخلية والقائمة.
15. تحسين الأداء النهائي.

## 16. معايير قبول النتيجة

- الموقع يعرض هوية وصل بوضوح من أول شاشة.
- التصميم غير تقليدي ويعتمد على فكرة الرحلة والمسار الذهبي.
- المحتوى مأخوذ من ملف العميل ومنظم بشكل احترافي.
- الصفحة تعمل بدون backend وبدون build tools.
- تجربة الجوال ممتازة وليست مكسورة.
- الحركات ناعمة وتخدم المعنى.
- الألوان مطابقة للهوية.
- CTA واضح ومتكرر بدون إزعاج.
- لا توجد كتل نصية طويلة غير مقروءة.
- الموقع قابل للتطوير لاحقا لإضافة صفحات أعمال، مدونة، أو نموذج تواصل.
